# Préparation de la session Zotero du jeudi

## 1. Cadrage rapide

**Événement** : PSF CREONS – 10 jours de formations autour de la science ouverte, de la reproductibilité et des partenariats scientifiques équitables.

**Contexte utile** : le programme CREONS vise une logique de formation de formateurs, avec un fort accent sur la science ouverte, la reproductibilité, la mutualisation des pratiques et la circulation des outils entre plusieurs antennes Nord-Sud.

## 2. Finalité pédagogique de la séance

L'objectif n'est pas de faire un tour complet de Zotero, mais de donner des routines simples, robustes et transmissibles.

À la fin de l'atelier, les participant·es doivent pouvoir :
1. importer correctement une référence ;
2. corriger les métadonnées essentielles ;
3. organiser une petite bibliothèque ;
4. partager une bibliographie ou une collection de travail ;
5. produire une citation et une bibliographie ;
6. comprendre ce que Zotero fait et ne fait pas ;
7. identifier les usages collectifs : groupe Zotero, règles minimales de nommage et de nettoyage ;
8. repartir avec une méthode qu'ils pourront refaire localement.

## 3. Message central à faire passer

**Zotero est une infrastructure légère de travail bibliographique collectif**.

Dans le cadre CREONS, il sert à :
- structurer une revue de littérature ;
- documenter proprement les sources ;
- partager des références entre collègues et antennes ;
- améliorer la traçabilité des citations ;
- soutenir des pratiques de recherche plus ouvertes et plus reproductibles.

On peut le présenter simplement ainsi : Zotero sert à **collecter**, **organiser** et **citer** des références de manière plus fiable, tout en permettant le **partage** entre collègues. Cela en fait un bon outil d'appui à une recherche plus ouverte, plus mutualisée et plus transmissible.

## 5. Exercice central

### Exercice fil rouge

**Construire une mini-bibliographie collaborative sur un thème CREONS**.

Sujet possible :
- science ouverte ;
- reproductibilité ;
- partenariats scientifiques équitables ;
- accès aux ressources bibliographiques.

### Consigne

Par petit groupe, les participant·es doivent :
1. importer **5 références** ;
2. corriger les champs essentiels ;
3. ajouter **un tag commun** ;
4. rédiger **une note courte** sur l'intérêt de la référence ;
5. repérer d'éventuels doublons ;
6. générer une bibliographie finale dans un style choisi.

### Sortie attendue

Chaque groupe repart avec :
- une collection propre ;
- quelques références fiabilisées ;
- une bibliographie exportée ;
- une méthode réutilisable localement.

## 6. Points pédagogiques à ne pas manquer

### Zotero n'est pas magique

Une référence importée n'est pas une référence validée. C'est sans doute le point le plus important de la séance.

Erreurs fréquentes à montrer :
- prénom et nom inversés ;
- titre en majuscules ou mal capitalisé ;
- type de document incorrect ;
- date ou revue manquante ;
- rapport importé comme page web.

### Collections ≠ dossiers physiques

Une même référence peut apparaître dans plusieurs collections sans être dupliquée. C'est très utile pour un travail collectif par thème, projet ou séance.

### Tags : rester simple

Mieux vaut une liste courte de tags communs qu'une accumulation de tags personnels inutilisables.

### PDF et stockage

Il faut distinguer :
- les **métadonnées** bibliographiques ;
- les **fichiers joints** PDF.

Dans un contexte multi-sites, cela permet d'ouvrir rapidement une discussion sur la synchronisation, la bande passante et les droits sur les fichiers.

### Groupes Zotero

Les groupes sont probablement l'entrée la plus pertinente pour CREONS : partage de bibliographies, capitalisation entre promotions, circulation de ressources entre antennes, et transmission locale après la formation.

## 7. Préparation à faire avant jeudi

### A. Supports et démonstration

Préparer :
- une **bibliothèque Zotero de démonstration** avec quelques erreurs volontaires ;
- un **document de test** pour les citations ;
- une **collection de démonstration** sur un thème lié à CREONS ;
- si possible un **groupe Zotero test**.

### B. Jeu de références de démonstration

Prévoir au moins :
- 1 article avec DOI bien reconnu ;
- 1 chapitre d'ouvrage mal importé ;
- 1 rapport institutionnel ;
- 1 page web ;
- 1 PDF aux métadonnées incomplètes ;
- 1 référence HAL ou OpenEdition ;
- 1 doublon.

Sources utiles à prévoir dans la démo :
- un import par DOI ;
- un import depuis HAL ou OpenEdition ;
- un import depuis un site d'éditeur ;
- un test depuis Google Scholar ou Cairn si cela fonctionne bien ;
- un PDF isolé pour montrer les limites de reconnaissance.

### C. Points techniques à vérifier

Vérifier avant la séance :
- Zotero installé ;
- connecteur navigateur fonctionnel ;
- insertion dans Word ou LibreOffice ;
- accès internet ;
- solution de secours si le connecteur ne marche pas ;
- export bibliographique simple ;
- éventuel compte Zotero si partage en groupe.

## 9. Les messages pédagogiques à marteler

1. **Une référence importée n'est pas une référence validée.**
2. **Zotero aide à organiser, mais n'automatise pas le jugement scientifique.**
3. **Une bibliothèque partagée a besoin de règles communes.**
4. **Le plus important n'est pas l'outil, mais la qualité des routines.**

## 10. Règles minimales pour le travail collectif

Je conseille de fixer dès l'atelier 5 règles simples :

1. Toujours vérifier auteur, date, titre, type de document.
2. Utiliser une liste courte de tags communs.
3. Éviter de multiplier les collections inutilement.
4. Fusionner les doublons régulièrement.
5. Ne pas corriger la bibliographie finale directement dans le traitement de texte avant la toute fin.

## 11. Plan minimal du support de cours

Le support peut rester très court, autour de 8 à 12 diapositives, organisé ainsi :

1. **Pourquoi Zotero ?**
2. **Installer** : application, connecteur, module de traitement de texte
3. **Collecter** : DOI, ISBN, sites éditeurs, HAL, PDF, pages web
4. **Nettoyer** : auteurs, dates, titres, type de document, DOI, URL
5. **Organiser** : collections, tags, notes, doublons
6. **Citer** : insertion dans Word/LibreOffice, changement de style, bibliographie automatique
7. **Collaborer** : groupes Zotero, règles communes, bibliothèque de projet
8. **Transmettre** : comment refaire cette formation localement

## 12. Fiche “bonnes pratiques” à distribuer

| Problème | Règle pratique |
|---|---|
| Import automatique incertain | Toujours vérifier les métadonnées |
| Trop de collections | Créer une structure simple par projet |
| Trop de tags | Définir 5 à 10 tags communs |
| Références dupliquées | Utiliser régulièrement la détection des doublons |
| Rapports mal cités | Vérifier institution, année, titre, URL |
| PDF sans métadonnées | Créer ou corriger la notice parent |
| Travail collectif | Utiliser un groupe Zotero et des règles partagées |
| Bibliographie finale | Générer depuis Zotero, corriger à la main seulement à la fin |

## 14. Risques à anticiper

### Risques fréquents
- connecteur navigateur non installé ;
- métadonnées très sales ;
- confusion entre collection et dossier ;
- temps perdu sur les styles bibliographiques ;
- comptes Zotero non créés ;
- différences Word / LibreOffice selon les machines.

### Réponse pratique
- garder un plan B par import DOI, ISBN ou ajout manuel ;
- limiter les styles à un seul pendant l'atelier ;
- faire une démonstration courte puis un exercice guidé ;
- privilégier la qualité de 5 références plutôt qu'une collecte massive.
